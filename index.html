Use Case
Building the "IT Accountability" Report in 3 Steps
1

Map Jira Fields to Salesforce

Admin creates custom fields on the Case object (Jira Status, Last Update, Target Date) and maps them in the Connector's binding configuration. One-time setup, takes ~15 minutes.

2

Enable Auto-Sync

Deploy a short Apex trigger and enable Auto Pull in the Connection settings. Every Jira status change, comment, or field update flows into Salesforce automatically.

3

Build Native Salesforce Reports

Use standard Salesforce Report Builder to create aging reports, bottleneck dashboards, and SLA tracking — all powered by live Jira data in native custom fields.

Paper Trail

Every Jira status change is recorded in Salesforce field history. Managers can audit exactly when a ticket moved, how long it sat, and who owned it — without ever opening Jira.

Efficiency Dividend

Eliminates the manual check-in cycle. Instead of chasing engineers for updates, Sarah's team sees live status in the same reports they already run. No new tools, no training gap.

Not an iFrame

Data lives in Salesforce custom fields — not a widget or embedded view. It's sortable, filterable, chartable, and works with Salesforce CRMA, Einstein, and any standard report type.

Field Mapping Specification

Recommended field mappings for IT responsiveness reporting. All fields support bidirectional sync and can be customized per project binding.

Jira Field Jira Type Sync Salesforce Field SF Type
Status Status ⟷ Jira_Status__c Picklist
Resolution Resolution ⟷ Jira_Resolution__c Picklist
Priority Priority ⟷ Jira_Priority__c Picklist
Updated Updated Date → Jira_Last_Updated__c Date/Time
Issue Key Key → Jira_Issue_Key__c Text
Summary Summary ⟷ Jira_Summary__c Long Text
Assignee Assignee → Jira_Assignee__c Text
Created Date Created → Jira_Created_Date__c Date/Time
Target End Date Due Date ⟷ Jira_Target_End_Date__c Date
Ticket Age formula field: Create a Salesforce formula field Jira_Ticket_Age__c = TODAY() - DATEVALUE(Jira_Created_Date__c) on the Case object. This gives you a real-time aging metric you can filter, sort, and chart in any Salesforce report — no custom code required.
Salesforce Report Types to Build

Ticket Aging Report

Report Type: Cases. Group by Jira_Status__c, sort by Jira_Ticket_Age__c descending. Filter to open tickets only. Highlights bottlenecks instantly.

IT Response Time

Report Type: Cases. Formula: days between Jira_Last_Updated__c and TODAY(). Group by Assignee. Surface tickets with no update in 7+ days.

SLA Compliance

Report Type: Cases. Compare Jira_Target_End_Date__c vs. resolution date. Chart % on-time by team, project, or priority. Schedule and email weekly.
